## 1. 概述
- **v1.2 (2026-02-24)**: 侧重于后端架构的健壮性提升，引入自动数据库迁移机制和 API 数据格式兼容层。

## 2. 后端架构增强详情

### 2.1 数据库自动迁移
- **机制引入**: 在 `backend/src/config/database.ts` 中实现了 `ensureColumn` 方法。
- **功能**: 系统启动时自动检测并修复缺失的数据库列（如 `tutorials` 表的 `image` 列），无需手动干预。
- **目的**: 解决开发环境与生产环境数据库结构不一致导致的 `SQLITE_ERROR`。

### 2.2 API 兼容性层
- **数据格式适配**: `BlogService` 现已支持接收字符串数组 (`string[]`) 格式的标签数据，并在 Service 层自动转换为数据库友好的字符串格式。
- **校验增强**: `validation-utils` 更新了对数组类型输入的校验逻辑，确保数据的完整性和安全性。
- **目的**: 消除因前端数据结构变更导致的 `ValidationError`，提高 API 的灵活性和健壮性。
